# Hypocamp workspace instructions

This is a private operational workspace for a user and their agents.

## Start of work

1. Read `map.md`.
2. Read `operations/agents.md`; register a stable agent ID if needed.
3. Read `operations/now.md`, current reports, and relevant lessons.
4. Capture a new request in `operations/inbox.md` before changing operational records.

## Working rules

- Keep verified facts separate from inferences, unknowns, and conflicts.
- Project repositories remain the source of truth for code and detailed technical state.
- Daily reports are the source of current operational context.
- Do not store secrets, tokens, credentials, private URLs, or unnecessary personal data.
- Do not silently change goals, durable decisions, access rules, or project stages.

## Finish

After meaningful work, update `operations/reports/YYYY-MM-DD/<agent_id>.md` with the objective, evidence, changes, checks, risks, and next safe action. Add a reusable lesson only when it is supported by evidence.
