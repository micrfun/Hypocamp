# Architecture

Hypocamp is an operational context layer. It stores the minimum shared model needed for people and agents to understand current work and continue it safely.

## External context memory

Hypocamp uses the hippocampus as a product metaphor, not a medical model. It should:

- consolidate fragmented facts, decisions, and handoffs into a durable working context;
- map projects, goals, commitments, and next actions so work can be navigated;
- preserve context through sources, confidence, verification dates, and reasons for change.

## Layers

```text
Sources: projects, Git, documents, calendars, messages, agents
    ↓
Canonical records: facts, states, goals, actions, decisions
    ↓
Protocols: triage, review, audit, handoff, synchronization
    ↓
Views: Markdown, dashboards, graphs, reports
```

Markdown files and Git history are canonical. Views are replaceable projections and must not become a second source of truth.

## Core record types

- `project` — bounded work with an intended outcome.
- `area` — an ongoing domain of responsibility.
- `goal` — a desired change of state.
- `task` — a concrete action.
- `commitment` — a promise, deadline, or expectation.
- `decision` — a durable rule or choice and its reason.
- `handoff` — a verifiable transfer of context.
- `agent-run` — the result of an agent session.

Every record should have a stable `id` and `type`. Important facts should include a source and verification date. Active records should expose a next action or an explicit waiting state.

## Lifecycle

Projects: `idea → incubating → active → paused → completed → archived`.

Tasks: `captured → clarified → ready → in_progress → waiting → done → cancelled`.

Data quality is separate from lifecycle: `confirmed`, `inferred`, `stale`, `unknown`, and `conflict`.

## Public and private deployments

The public repository contains reusable structure and documentation. A private deployment contains user records and operational context. The public core must never be generated by copying a private workspace without an explicit sanitization and review step.

## Derived analysis

Graph and dashboard tools can index canonical records and reveal relationships, but they remain derived views. Any cross-project relationship that affects decisions should be recorded explicitly in canonical records.
