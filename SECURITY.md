# Security and privacy

Hypocamp is intended to help manage sensitive operational context, so privacy boundaries are part of the design.

## Do not publish

- personal records or identifying information;
- credentials, tokens, private URLs, or private network details;
- real project histories, commitments, or agent transcripts;
- local filesystem paths or deployment-specific configuration.

## Reporting an issue

Do not open a public issue for a suspected credential leak or private-data exposure. Contact the project maintainers privately through the GitHub account that owns this repository and include only the minimum necessary detail.

If the report is about the public documentation or a reproducible design flaw, a public issue is appropriate after confirming that it contains no sensitive information.
