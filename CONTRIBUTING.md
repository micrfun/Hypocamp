# Contributing to Hypocamp

Hypocamp is being developed as a reusable methodology and tool foundation for people and AI agents.

## Before opening a change

- Read `README.md` and the relevant documents in `docs/`.
- Keep examples synthetic and safe to publish.
- Explain whether the change affects the record model, lifecycle, agent protocol, or public/private boundary.
- Prefer small, focused pull requests.

## Documentation changes

Use clear international English. Define new terms before using them, preserve stable identifiers, and distinguish confirmed facts from proposals or inferences.

## Model changes

Changes to record types, lifecycle states, evidence fields, or privacy boundaries require:

1. a short rationale;
2. compatibility notes;
3. an example or validation rule;
4. an update to the relevant architecture or protocol document.

## Privacy review

Before submitting, check that the change contains no personal data, private project records, secrets, private URLs, local paths, or agent transcripts.
